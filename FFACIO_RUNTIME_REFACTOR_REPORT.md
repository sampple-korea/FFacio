# FFacio Runtime Demo 정렬·보안 보강 구현 보고서

## 적용 버전

- 앱 버전: `0.8.2-secure-smartthings-tracking`
- 저장 정책·레코드 스키마: 8
- SmartThings 설정 스키마: 2
- 카메라: Runtime Demo와 동일한 Fotoapparat 2.7.0 NV21 흐름
- 얼굴 엔진: FFacio Runtime Binder 경로만 사용
- 등록: 1200ms 최적 원본 프레임 재검사 후 대표 템플릿 1개
- 인증 안정화: 사용자 요구에 따라 1프레임

## 실제 문제 원인과 수정

이전 FFacio는 Runtime 연결 자체는 성공했지만 Runtime Demo와 카메라 입력 경로가 달랐습니다. CameraX의 `YUV_420_888`을 plane·stride·crop 기반으로 직접 NV21로 재조립하고 별도 회전 처리를 했고, 이 차이가 눈·자세·랜드마크 값이 Demo와 다르게 나오는 가장 큰 원인이었습니다.

현재 구현은 Demo의 Fotoapparat AAR, NV21 원본 프레임, `frame.rotation`, 전면 EXIF 미러링, `CenterCrop` 좌표 변환을 그대로 사용합니다. 등록은 1280×720, 인증은 640×480을 우선하며 정확한 크기가 없을 때에는 종횡비를 우선해 가장 가까운 해상도를 선택합니다.

## 등록 구현

- 가장 큰 얼굴 한 명만 선택
- 얼굴 좌표·68점 랜드마크·크기·중앙·품질·밝기·심한 자세·라이브니스·선택형 가림 검사
- 눈 감김·입 벌림 값은 진단용으로만 사용
- 1200ms 동안 품질이 가장 좋은 원본 NV21 프레임만 독립 소유
- 완료 시 원본 프레임을 다시 변환·검출·판정한 뒤 템플릿을 한 번만 추출
- 기존 사용자 전원과 비교하며 각 비교는 일시 오류를 한 번 재시도
- 한 건이라도 비교 실패·템플릿 불일치·범위 밖 점수가 있으면 저장 중단
- 최고 유사도 0.80 이상이면 동일 얼굴로 보고 등록 차단
- 정규화한 표시 이름의 제어문자·길이·대소문자 무시 중복도 차단

## 인증 구현

Runtime 라이브니스 0.70, 품질 0.50, 얼굴 면적 3%, 심한 자세(yaw 35°, pitch/roll 30° 초과), 선택형 가림, 전체 사용자 비교 완료, 최고 유사도 0.80, 1위·2위 차이 0.03을 적용합니다. Runtime의 확률·유사도 값은 0.0~1.0 범위를 벗어나면 오류로 처리합니다.

능동 고개 돌리기 챌린지, 정면·좌·우 5장 등록, 자세별 유지, 샘플 응집도·대표 샘플·보조 지지 판정은 제거했습니다.

## 실시간 추적과 UI

- Demo의 oriented bitmap 좌표를 `CenterCrop`으로 화면에 매핑한 실제 얼굴 경계 사각형
- 가장 큰 얼굴을 따라 이동하는 Face ID 스타일 링과 90ms 사각형 보간
- 둥근 카메라 카드·유리 계열 테두리·iOS 계열 상태색
- 안내 문구는 하단 상태 패널 한 곳에만 표시
- 카메라가 켜진 동안 내부에는 얼굴 추적 그래픽과 관리자 인증 취소 조작만 표시

## SmartThings 도어락

임의 relay URL 연동은 제거했습니다. 고정된 SmartThings API host에서 기기 `lock` capability 상태를 조회하고, 인증 성공 시 `/devices/{deviceId}/commands`에 `main` 컴포넌트의 `unlock` 명령을 전송합니다.

문 열림은 설정 활성화, 정상 Device ID/token, Runtime 라이브니스 켜짐, 현재 프레임 실제 얼굴 통과, 전체 사용자 비교 완료, 유사도·후보 격차를 모두 만족할 때만 실행합니다. access token은 Android Keystore AES-GCM으로 암호화하고 저장 키 이름을 AAD로 결합합니다. redirect를 금지하고 입력·응답 크기를 제한하며, 중간 저장 실패 시 Device ID·암호문·활성화 플래그를 함께 롤백합니다.

## 데이터와 메모리 보호

정책 버전 8에서 이전 얼굴 등록 데이터와 판정 옵션을 폐기합니다. 등록 프레임, 추출 템플릿, 저장 스냅샷과 비교 후보는 소유권을 분리하고 실패·취소·교체·오래된 비동기 결과에서 배열을 0으로 덮습니다. 스크린샷 방지, 앱 백업 제외, cleartext 차단, Runtime signature 권한을 유지합니다.
