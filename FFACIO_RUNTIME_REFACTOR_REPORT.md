# FFacio Runtime Demo 정렬 구현 보고서

## 적용 버전

- 앱 버전: `0.7.1-runtime-demo-parity-ios`
- 저장 정책·레코드 스키마: 7
- 카메라: Runtime Demo와 동일한 Fotoapparat 2.7.0 NV21 흐름
- 얼굴 엔진: FFacio Runtime Binder 경로만 사용
- 등록: 최적 원본 프레임 최종 재검사 후 대표 템플릿 1개
- 인증 안정화: 1프레임

## 실제 문제 원인과 수정

이전 FFacio는 Runtime 연결 자체는 성공했지만 Runtime Demo와 카메라 입력 경로가 달랐습니다. CameraX의 `YUV_420_888`을 plane·stride·crop 기반으로 직접 NV21로 재조립하고 별도 회전 처리를 했습니다. Runtime Demo는 Fotoapparat가 제공하는 tightly packed NV21와 `frame.rotation`을 그대로 사용합니다.

얼굴 검출은 되더라도 YUV 크로마 배치, 센서 회전 또는 전면 미러링이 다르면 눈 감김·yaw·pitch·roll·랜드마크처럼 얼굴 내부 구조에 의존하는 값이 비정상적으로 나올 수 있습니다. 따라서 임계값만 완화하지 않고 입력 파이프라인을 Demo 방식으로 교체했습니다.

- Demo의 `fotoapparat-2.7.0.aar`를 바이트 동일하게 포함
- CameraX와 수동 NV21 변환 코드 제거
- Demo orientation 8개 매핑 이식
- 등록 1280×720, 인증 640×480 우선 해상도
- 전면 카메라 우선, 없으면 후면 카메라 대체
- `CenterCrop` 미리보기와 oriented bitmap 좌표를 같은 방식으로 매핑

## 등록 구현

등록은 Demo `CaptureActivityKt`, `RegistrationPipeline`, `RegistrationInspector`, `CaptureView.getROIRect1()`을 기준으로 구현했습니다. 사용자 요구에 따른 차이는 여러 얼굴에서 가장 큰 얼굴을 선택하는 것뿐입니다.

유효 프레임은 1200ms 동안 연속이어야 하며, 품질이 더 좋은 원본 NV21 프레임만 보관합니다. 완료 후에는 그 프레임을 다시 변환하고 가장 큰 얼굴을 다시 검출하며 모든 등록 조건을 재검사한 뒤 템플릿을 한 번만 추출합니다. 따라서 안정화 중 보았던 얼굴과 최종 저장 얼굴이 달라지거나 마지막 프레임 조건이 무너진 경우 저장하지 않습니다.

기존 사용자와의 비교는 전체 후보를 검사해 가장 높은 유사도만 경고합니다. 한 사용자 비교가 실패해도 전체 등록을 실패시키지 않습니다. 경고 후 별도 등록은 허용합니다.

## 인증 구현

인증은 Demo의 라이브니스·품질·최소 면적 검사를 적용한 뒤 Runtime 템플릿 비교를 수행합니다. 등록에서 사용하는 눈·입·자세 값은 인증을 막지 않습니다. 최고 점수 0.80과 2위 차이 0.03을 적용하며 안정화는 사용자 요구대로 한 프레임입니다.

## 데이터와 메모리 보호

정책 버전 7에서 이전 얼굴 등록 데이터와 얼굴 판정 옵션을 폐기합니다. 새 등록은 사용자당 템플릿 하나만 저장합니다. 등록 프레임, 추출 템플릿, 저장 스냅샷과 비교 후보는 소유권을 분리하고 실패·취소·교체·오래된 비동기 결과에서 배열을 0으로 덮습니다. 손상된 개별 사용자 레코드는 정상 사용자 로딩을 막지 않고 제외합니다.

## 검증 범위

- Runtime Demo와 Fotoapparat AAR SHA-256 동일성
- Demo 설정값·orientation·검출 옵션·중앙 정사각형 ROI 대조
- AIDL 3개, Parcel 2개, FaceBox·FaceDetectionParam의 바이트 동일성
- 전체 XML 파싱, Kotlin 구문 트리 검사, 정적 fail-closed 감사
- 실제 `RuntimeDemoPolicy.kt`와 최종 등록 프레임 소유권 코드를 Kotlin/JVM 17로 컴파일하고 39개 실행 검증
- 89개 JUnit 회귀 테스트 소스의 구조·수량 검사
- 최종 ZIP 재해제 후 같은 검증 반복

전체 Gradle 앱 빌드는 실행 환경에서 Gradle 8.13 배포본을 내려받지 못해 제외합니다. 이는 최종 보고서에서 빌드 성공으로 표시하지 않습니다.
