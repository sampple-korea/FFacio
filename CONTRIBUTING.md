# Contributing

Thanks for helping improve this project.

## Before opening an issue or pull request

- Read the README and search existing issues first.
- Keep changes focused and explain the problem they solve.
- Do not commit credentials, tokens, private configuration files, personal data, or production data.
- Update documentation and tests when the change affects behaviour.

## Pull requests

1. Create a focused branch from the default branch.
2. Describe the motivation, implementation, and validation performed.
3. Keep unrelated formatting and generated-file changes out of the pull request.
4. Make sure any new dependencies and third-party assets have compatible licenses.
