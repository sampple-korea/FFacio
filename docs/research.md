# Runtime 판정 정책 메모

FFacio Android 0.8.2의 얼굴 입력과 템플릿 생성·비교 흐름은 제공된 FFacio Runtime Demo 구현을 기준으로 정리했습니다. 앱은 Runtime 템플릿을 해석하거나 자체 임베딩으로 대체하지 않습니다.

핵심 수치는 식별 기준 0.80, 1위·2위 차이 0.03, 라이브니스 0.70, 품질 0.50, 선택형 가림 기준 0.50, 등록 안정화 1200ms, 분석 간격 180ms입니다. 사용자가 요구한 대로 인증 안정화는 1프레임입니다. 이로 인한 완화분은 Runtime 라이브니스, 품질·최소 얼굴 면적, 심한 고개 회전 차단, 전체 사용자 비교 완료, 1위·2위 격차와 SmartThings 단일 실행 잠금으로 보완합니다.

눈 감김과 입 열림 속성은 Runtime Demo와 같은 검출 요청에 포함하지만, 실제 얼굴에서 잘못된 안내가 반복됐기 때문에 등록·인증 차단에는 사용하지 않습니다. 자세도 작은 흔들림은 허용하되 yaw 35도, pitch 30도, roll 30도를 초과하는 명백한 회전·기울기만 차단합니다. 선택형 얼굴 가림 검사를 켜면 등록과 인증에 모두 적용됩니다.

등록은 1.2초 동안 가장 품질이 좋은 원본 Runtime Demo 카메라 프레임을 보관한 뒤 재검출·재판정하고 대표 Runtime 템플릿 하나만 추출합니다. 새 템플릿은 기존 사용자 전원과 비교하며, 일시적 Binder 오류는 한 번 재시도합니다. 한 건이라도 비교가 끝나지 않거나 최고 유사도가 0.80 이상이면 등록을 차단합니다.

인증은 사용자당 대표 템플릿 하나를 비교하고 여러 얼굴이 보이면 가장 큰 얼굴만 처리합니다. 능동 동작 챌린지, 다각도 등록, 샘플 간 일관성 검사와 보조 샘플 지지 판정은 사용하지 않습니다.

이 기준은 실제 배치 환경의 카메라, 거리, 조명, Runtime 모델 버전과 사용자 집단에 따라 오인식·거부율을 따로 측정해야 합니다. 관리자 판정 로그에는 최고 점수·2위 점수·거부 이유를 기록하지만 공개 운용 화면에는 생체 점수나 과거 사용자 이름을 노출하지 않습니다.

Runtime이 반환하는 유사도·라이브니스·품질·가림 값은 정상 계약 범위인 0.0~1.0에서만 사용합니다. 범위를 벗어난 값과 비유한수는 잘못된 고득점으로 인증되는 것을 막기 위해 오류로 처리합니다.
